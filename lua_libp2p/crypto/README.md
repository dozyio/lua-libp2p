# lua_libp2p.crypto

Crypto module entrypoint.

Public keys used for PeerId derivation are encoded as libp2p `PublicKey`
protobuf bytes with deterministic field order and minimal varints.

Supported identity key types are Ed25519, RSA, ECDSA, and Secp256k1. Ed25519
protobuf public keys store the raw 32 byte public key; Ed25519 private keys
are stored locally as the raw 64 byte `[private][public]` key. RSA public keys
use DER-encoded PKIX/SPKI bytes, ECDSA public keys use DER-encoded ASN.1
public key bytes, and Secp256k1 public keys use compressed or uncompressed
Bitcoin EC point bytes.

`lua_libp2p.crypto.keys` provides key generation, signing, verification, and
host-identity helpers. RSA, ECDSA, and Secp256k1 private keys generated by
this module are retained locally as PEM for signing; public wire encoding
remains spec-compatible.

No public LuaLS symbols are documented for this module yet.
